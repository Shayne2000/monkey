Work status - RTSP motion gate + vehicle model pipeline

Current branch:
- codex/rtsp-relay-starter

What was added in this round:
- robust_rtsp_relay.py now has a VehicleModelPipeline skeleton using OpenCV DNN.
- The intended flow is:
  1. Read RTSP frames using the existing Jetson/GStreamer reader.
  2. Run lightweight 2-frame motion detection.
  3. Expand motion boxes into larger square crops for higher recall.
  4. Run YOLO only on motion crops.
  5. Run the classifier on detected vehicle crops.
  6. Write vehicle_log-shaped JSONL events with vehicle_type and brand when available.
- The model paths are configurable:
  --yolo-model models/yolo.onnx
  --classifier-model models/classifier.onnx
  --labels models/labels.json
- Models can be disabled with:
  --no-models
